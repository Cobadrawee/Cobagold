# Connect Wallet QR code not showing

When you tap a wallet in the Connect Wallet modal, a QR code should appear so you can scan it with your phone. If you only see a spinner or nothing, do the following.

## 1. Set a valid Reown Project ID (required)

The QR is generated by WalletConnect/Reown. They need a **project ID** from their dashboard.

1. Go to **https://dashboard.reown.com**
2. Sign up or log in
3. Create a project (or use an existing one)
4. Copy the **Project ID**
5. In your project root, create a `.env` file (or edit it) and add:

   ```env
   VITE_WALLETCONNECT_PROJECT_ID=your_actual_project_id_here
   ```

6. Restart the dev server (`npm run dev`)

Without a valid project ID, the relay never returns a connection URI, so no QR can be shown.

## 2. Allow your domain in the dashboard (for production)

In the Reown dashboard, open your project and add your site’s domain (e.g. `https://cobagold.com`) to the allowed origins/list. Localhost is usually allowed by default.

## 3. Network / firewall

The app talks to `wss://relay.walletconnect.org`. If you’re on a locked-down network (corporate VPN, strict firewall, or restricted country), the WebSocket can fail and the QR never appears. Try another network or disable VPN to test.

## 4. Debug in the browser

To see connection errors in the console:

1. In `.env` add: `VITE_APPKIT_DEBUG=1`
2. Restart the dev server and open the browser console (F12 → Console)
3. Click Connect Wallet and then a wallet; check for red errors

That will show any relay or project ID errors.
